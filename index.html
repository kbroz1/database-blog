5 Types of SQL Commands

  1. Data Definition Language (DDL)

    DDL changes the structure of the table. With DDL, you can CREATE, ALTER, DROP, or TRUNCATE a table. Those words in all caps are examples DDL commands.

  2. Data Manipulation Language(DML))

    DML commands are used change the database. With DML, you can INSERT, UPDATE, and DELETE data from the table. Those words in all caps are examples DML commands.

  3. Data Control Language (DCL)

    DCL commands are used to grant and take back authority from any database user. With DCL, you can GRANT and REVOKE the privileges of a database user. Those words in all caps are examples DCL commands.

  4. Transaction Control Language (TCL)

    This command is used to manage the changes made by DML statements. TCL allows the statements to be grouped together into logical transactions. With TCL, you can COMMIT, ROLLBACK and SAVEPOINT changes paired with DML commands. Those words in all caps are examples DCL commands.

  5. Data Query Language (DQL)

    DQL commands are used to get data from the database. With DQL, you can SELECT the data that you want from the table. SELECT is the only command for DQL.

6 Steps for Setting Up a Database

DEFINE

Define the Purpose of the Database
  • Decide the purpose of the database, gather data, construct table
  • Divide into subject based tables
  • Create a sample form

GATHER

Gather Data, Organize in tables and Specify the Primary Keys
  • Input your data into the tables
  • Choose which data piece will be your Primary Key
  • Make sure to use good naming conventions

RELATE

Create relationships among tables
  • Identify how your tables connect with each other
  • Identify shared fields across tables
  • Make connections

REFINE AND NORMALIZE

Apply normalization rules

What is normalization?

Normalization is the process of organizing data in a database. This includes creating tables and establishing relationships between those tables according to rules designed both to protect the data and to make the database more flexible by eliminating redundancy and inconsistent dependency.

ENFORCE DOMAIN INTEGRITY

Use appropriate data types
  • Enforce rules and constraints to make sure all data is consistent
  • Make sure related data is in a shared format

STYLE

Define or adopt a style
  • Establish analyst-friendly naming conventions
  • Set up clear and consistent guidelines for all the stored information
